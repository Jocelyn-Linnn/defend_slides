Inference Time Pipeline Preprocess
1
Load & Preprocess
  • Lemmatize entity tokens, strip numbers and punctuation
  • Assign feature types OBS-DP-E / OBS-DA-E / OBS-U-E
2
Build Supernodes
  • Run BFS to find connected entity components and merge them into Supernodes with subgraph structure
3
Identify Core Node
  • Pick the one entity that best represents the Supernode's finding + presence state — priority: OBS-DP-E › OBS-DA-E › OBS-U-E
Why
Same Supernode text (e.g. "opacity") can carry different presence states — definitely-present vs. definitely-absent — and these map to different SUGGESTIVE_OF paths in the graph. The core node's text + feature pinpoints exactly which one to retrieve.
4
Embed & Export
Use Clinical-BigBird to encode each Supernode's text into an embedding stored in the Supernode.
