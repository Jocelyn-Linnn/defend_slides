Retrieve Method 1 · Single-Layer Expansion with Fixed Top-K
1
Match Corenode
  • Construct entity_id = text_feature.
  • Match to the corresponding Entity node in the DB as the retrieval anchor.
2
Embedding Match
  • Query DB supernodes sharing the same core entity.
  • Compute cosine similarity and select the best match.
  • If best sim < 0.95, skip — no relevant entry exists in DB.
3
L1 Outgoing Traversal
  • From matched DB supernode, follow all outgoing SUGGESTIVE_OF edges to retrieve L1 neighbors with edge weights.
4
Top-5 Selection & Union
  • Sort L1 neighbors by weight, keep top 5.
  • Union across all supernodes.
  • Fetch stored questions as output.
